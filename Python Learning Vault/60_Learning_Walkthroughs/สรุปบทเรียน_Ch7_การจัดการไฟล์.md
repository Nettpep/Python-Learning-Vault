## 🎯 สรุปหนึ่งบรรทัด

ก้าวจาก "ข้อมูลในหน่วยความจำที่หายเมื่อปิดโปรแกรม" สู่การทำงานกับ **ไฟล์บนดิสก์** (Secondary Storage) ผ่าน **File Handle** และเข้าใจบทบาทของอักขระ **`\n`** (ขึ้นบรรทัดใหม่)

---

## 1. ก้าวข้ามจาก "จินตนาการ" สู่ "โลกจริง"

| ก่อนหน้า | ตอนนี้ |
|----------|--------|
| เรียนรู้การตั้งค่าตัวแปร \(x = 3\) หรือการวนลูป — **ข้อมูลหายไปทันทีเมื่อปิดโปรแกรม** (เหมือนต้นไม้ล้มในป่าโดยไม่มีใครเห็น) | เริ่มทำงานกับ **Secondary Storage** (หน่วยเก็บข้อมูลถาวร) เช่น ฮาร์ดดิสก์ — **เก็บข้อมูลได้ตลอดไปแม้ปิดเครื่อง** |

---

## 2. ไฟล์ข้อความ (Flat Text Files)

- ไฟล์ที่ใช้งานคือ **ไฟล์ข้อความบริสุทธิ์** (เช่น `.txt` หรือ `.py`) ไม่ใช่ไฟล์ที่มีรูปแบบซับซ้อนอย่าง PDF หรือ Word
- **โครงสร้าง:** ไฟล์คือ **ลำดับของตัวอักษร** ที่แบ่งเป็นบรรทัดๆ โดยมี **อักขระพิเศษที่มองไม่เห็น** คอยกำกับจุดจบของแต่ละบรรทัด

---

## 3. ตัวจัดการไฟล์ (File Handle)

การจะทำงานกับไฟล์ ต้องสั่ง **`open()`** ก่อนเสมอ:

| รายการ | คำอธิบาย |
|--------|----------|
| **ฟังก์ชัน** | `handle = open(filename, mode)` |
| **Handle ไม่ใช่ข้อมูล** | มันคือ **"สะพาน"** หรือ **"ช่องทาง"** ระหว่างโปรแกรมกับไฟล์บนดิสก์ (ยังไม่ได้อ่านข้อมูลเข้า memory ทั้งไฟล์) |
| **โหมด** | ถ้าไม่ระบุ จะถือว่าเป็นโหมดอ่าน `'r'` |
| **ข้อควรระวัง** | ถ้าสั่ง `open()` ไฟล์ที่**ไม่มีอยู่จริง** Python จะแจ้งข้อผิดพลาด (Traceback) ทันที |

---

## 4. อักขระขึ้นบรรทัดใหม่ (The Newline Character: `\n`)

หัวใจของการจัดการไฟล์ข้อความ:

| รายการ | คำอธิบาย |
|--------|----------|
| **สัญลักษณ์** | ใน Python เขียนแทนด้วย **`\n`** |
| **ความจริง** | แม้เราจะเห็นเป็น 2 ตัวอักษร (backslash + n) ในโค้ด แต่ในระบบคอมพิวเตอร์มันคือ **อักขระตัวเดียว** |
| **หน้าที่** | ทำหน้าที่เหมือนปุ่ม **Enter** บนคีย์บอร์ด — สั่งให้ย้ายไปเริ่มต้นบรรทัดถัดไป |
| **Whitespace** | `\n` ถือเป็น **ช่องว่าง** (มองไม่เห็นด้วยตาเปล่า) แต่มีผลต่อการนับจำนวนอักขระและการแสดงผล |

## 5. กระบวนการทำงานพื้นฐาน

| ขั้นตอน | ความหมาย |
|---------|----------|
| **Open** | เปิดสะพานเชื่อมต่อ (ได้ File Handle) |
| **Read / Write** | อ่านข้อมูลผ่านสะพาน หรือเขียนข้อมูลลงไป |
| **Close** | ปิดสะพานเมื่อใช้งานเสร็จ (เพื่อคืนทรัพยากรให้ระบบ) — หรือใช้ `with` ให้ปิดอัตโนมัติ |

## 6. ตัวอย่างสำคัญ: อ่านไฟล์แล้วแปลงเป็นตัวพิมพ์ใหญ่

```python
fname = input("Enter file name: ")
try:
    fhand = open(fname)
except:
    print("File cannot be opened:", fname)
    quit()
for line in fhand:
    line = line.rstrip().upper()
    print(line)
```

### ภาพรวมของโปรแกรม

โปรแกรมนี้ทำงาน 3 อย่างต่อเนื่องกัน:

1. รับชื่อไฟล์จากผู้ใช้
2. พยายามเปิดไฟล์ และดักข้อผิดพลาดถ้าเปิดไม่ได้
3. อ่านไฟล์ทีละบรรทัด แล้วแปลงข้อความในแต่ละบรรทัดให้เป็น **ตัวพิมพ์ใหญ่ทั้งหมด** ก่อนพิมพ์ออกหน้าจอ

### อธิบายทีละบรรทัด

| โค้ด | อธิบายละเอียด |
|------|----------------|
| `fname = input("Enter file name: ")` | รับชื่อไฟล์จากผู้ใช้ เช่น `mbox.txt` แล้วเก็บไว้ในตัวแปร `fname` ซึ่งตอนนี้ยังเป็นแค่ **ข้อความ (string)** ไม่ใช่ไฟล์ที่เปิดแล้ว |
| `try:` | เริ่มบล็อกที่ Python จะลองทำคำสั่งด้านใน ถ้ามี error เกิดขึ้น จะกระโดดไปทำใน `except` |
| `fhand = open(fname)` | พยายามเปิดไฟล์ตามชื่อที่ผู้ใช้พิมพ์ ถ้าสำเร็จจะได้ **file handle** มาเก็บใน `fhand` เพื่อใช้เข้าถึงข้อมูลในไฟล์ |
| `except:` | ถ้า `open(fname)` ล้มเหลว เช่น พิมพ์ชื่อไฟล์ผิด ไฟล์ไม่มีอยู่ หรือไม่มีสิทธิ์เปิดไฟล์ Python จะมาทำส่วนนี้แทน |
| `print("File cannot be opened:", fname)` | แจ้งผู้ใช้ว่าเปิดไฟล์ไม่ได้ พร้อมบอกชื่อไฟล์ที่พยายามเปิด |
| `quit()` | หยุดโปรแกรมทันที เพราะถ้าไม่มีไฟล์ให้เปิด ก็ไม่สามารถอ่านต่อได้ |
| `for line in fhand:` | วนลูปอ่านไฟล์ **ทีละบรรทัด** โดยในแต่ละรอบ `line` จะเป็นข้อความ 1 บรรทัดจากไฟล์ |
| `line = line.rstrip().upper()` | ประมวลผลข้อความ 2 ขั้นตอน: `rstrip()` ตัดช่องว่างด้านขวาและ `\n` ท้ายบรรทัดออกก่อน จากนั้น `upper()` แปลงตัวอักษรทั้งหมดให้เป็นตัวพิมพ์ใหญ่ |
| `print(line)` | แสดงผลบรรทัดที่ถูกทำความสะอาดและแปลงเป็นตัวพิมพ์ใหญ่แล้ว |

### ทำไมต้องใช้ `rstrip()` ก่อน `print()`

เวลาวนอ่านไฟล์ด้วย `for line in fhand:` ข้อความใน `line` มักจะติด `\n` มาด้วย เพราะแต่ละบรรทัดในไฟล์จบด้วยอักขระขึ้นบรรทัดใหม่

ถ้าเรา `print(line)` ตรงๆ:

- `line` มี `\n` อยู่แล้ว 1 ตัว
- `print()` ก็เติมการขึ้นบรรทัดใหม่ให้อีก 1 ครั้ง
- ผลลัพธ์จึงมักดูเหมือนมี **บรรทัดว่างคั่น**

`rstrip()` จึงถูกใช้เพื่อตัด `\n` ท้ายบรรทัดออกก่อน ทำให้แสดงผลได้สวยและไม่เว้นบรรทัดเกิน

### ทำไมใช้ `upper()`

`upper()` เป็นเมธอดของสตริง ใช้เปลี่ยนตัวอักษรภาษาอังกฤษทั้งหมดให้เป็นตัวพิมพ์ใหญ่ เช่น:

- `"hello".upper()` ได้ `"HELLO"`
- `"From stephen".upper()` ได้ `"FROM STEPHEN"`

ดังนั้นโปรแกรมนี้ไม่ใช่แค่อ่านไฟล์ แต่ยังเป็นตัวอย่างของการนำความรู้เรื่อง **สตริง** มาใช้กับข้อมูลที่อ่านจากไฟล์ด้วย

### ลำดับการไหลของข้อมูล

`ชื่อไฟล์จากผู้ใช้` → `open()` → `ได้ file handle` → `อ่านทีละบรรทัด` → `ตัด \n` → `แปลงเป็นตัวพิมพ์ใหญ่` → `print()`

### จุดเรียนรู้สำคัญจากตัวอย่างนี้

- `input()` ใช้รับข้อมูลจากผู้ใช้
- `open()` ใช้เปิดไฟล์ และถ้าเปิดสำเร็จจะได้ file handle
- `try/except` ช่วยให้โปรแกรม **ไม่พังด้วย traceback** เมื่อเกิดปัญหา
- การอ่านไฟล์ด้วย `for line in fhand:` เป็นวิธีที่นิยมและประหยัดหน่วยความจำ
- ข้อมูลแต่ละบรรทัดที่อ่านมาเป็น **string**
- `rstrip()` และ `upper()` คือการประมวลผลข้อความหลังจากอ่านไฟล์

### ตัวอย่างผลลัพธ์

ถ้าไฟล์มีข้อความ:

```text
Hello there
Python file handling
```

โปรแกรมจะแสดง:

```text
HELLO THERE
PYTHON FILE HANDLING
```

### ข้อสังเกตเพิ่มเติม

- โค้ดนี้ไม่ได้ใช้ `with open(...) as f:` แต่ก็ยังทำงานได้
- ในงานจริง มักนิยมใช้ `with` เพราะช่วยปิดไฟล์ให้อัตโนมัติเมื่อใช้งานเสร็จ
- `except:` แบบกว้างใช้งานได้ในบทเรียนเริ่มต้น แต่ในงานจริงควรระบุชนิด error ให้ชัดขึ้นเมื่อเป็นไปได้

## 7. ตัวอย่างสำคัญ: อ่านไฟล์เพื่อหาค่าเฉลี่ยของ `X-DSPAM-Confidence`

```python
# 1. รับชื่อไฟล์จากผู้ใช้
fname = input("Enter file name: ")

# 2. เปิดไฟล์ด้วย try/except เพื่อความปลอดภัย
try:
    fh = open(fname)
except:
    print("File cannot be opened:", fname)
    quit()

count = 0      # ตัวนับจำนวนบรรทัดที่พบ
total_val = 0  # ตัวแปรเก็บผลรวมของค่า Confidence

# 3. วนลูปอ่านไฟล์ทีละบรรทัด
for line in fh:
    # 4. ตรวจสอบว่าบรรทัดขึ้นต้นด้วยข้อความที่กำหนดหรือไม่
    if not line.startswith("X-DSPAM-Confidence:"):
        continue

    # 5. หากเจอ ให้เพิ่มจำนวนนับ
    count = count + 1

    # 6. ค้นหาตำแหน่งของตัวเลขและสกัดออกมา
    pos = line.find(':')
    str_val = line[pos+1:]

    # 7. แปลงเป็นทศนิยมและสะสมผลรวม
    val = float(str_val.strip())
    total_val = total_val + val

# 8. คำนวณค่าเฉลี่ยและแสดงผล
if count > 0:
    average = total_val / count
    print("Average spam confidence:", average)
else:
    print("No lines found.")
```

### โปรแกรมนี้ทำอะไร

โปรแกรมนี้ใช้สำหรับ:

1. เปิดไฟล์ข้อความ
2. อ่านไฟล์ทีละบรรทัด
3. เลือกเฉพาะบรรทัดที่ขึ้นต้นด้วย `"X-DSPAM-Confidence:"`
4. สกัดค่าตัวเลขที่อยู่หลัง `:`
5. แปลงค่าที่สกัดได้ให้เป็น `float`
6. รวมค่าทั้งหมดและหารด้วยจำนวนบรรทัดที่พบ เพื่อหาค่าเฉลี่ย

นี่เป็นตัวอย่างที่ดีมากของกระบวนการแบบ:

`Read` → `Search` → `Extract` → `Convert` → `Accumulate` → `Average`

### อธิบายทีละส่วนอย่างละเอียด

| โค้ด | อธิบายละเอียด |
|------|----------------|
| `fname = input("Enter file name: ")` | รับชื่อไฟล์จากผู้ใช้ และเก็บเป็นสตริงในตัวแปร `fname` |

| `str_val = line[pos+1:]` | ตัดข้อความตั้งแต่หลัง `:` ไปจนจบบรรทัด เช่น ได้ `" 0.8475\n"` |

| `total_val = total_val + val` | นำค่าที่ได้ไปสะสมรวมไว้ |
| `if count > 0:` | ป้องกันการหารด้วยศูนย์ ถ้าไม่พบบรรทัดที่ตรงเงื่อนไขเลยจะไม่คำนวณค่าเฉลี่ย |
| `average = total_val / count` | เอาผลรวมหารด้วยจำนวนข้อมูล เพื่อหาค่าเฉลี่ย |

| `print("No lines found.")` | แสดงข้อความกรณีไม่มีบรรทัดที่ตรงเงื่อนไขเลย |

### เจาะลึกแนวคิดสำคัญในโปรแกรมนี้

#### 1. การกรองข้อมูลด้วย `startswith()`

ไฟล์จริงมักมีหลายบรรทัดและไม่ได้ต้องการทุกบรรทัด โปรแกรมนี้จึงใช้:

```python
if not line.startswith("X-DSPAM-Confidence:"):
    continue
```

ความหมายคือ:

- ถ้าบรรทัดนี้ **ไม่ได้ขึ้นต้น** ด้วยข้อความเป้าหมาย
- ก็ **ข้ามบรรทัดนี้ไปเลย**
- เหลือเฉพาะบรรทัดที่เราต้องการประมวลผลจริงๆ

ตัวอย่าง:

- `"X-DSPAM-Confidence: 0.8475"` ผ่านเงื่อนไข
- `"From: someone@example.com"` ไม่ผ่านเงื่อนไข
- `"Subject: Hello"` ไม่ผ่านเงื่อนไข

#### 2. การสกัดข้อมูลด้วย `find()` และ slicing

เมื่อเจอบรรทัดที่ต้องการแล้ว ขั้นต่อไปคือดึงเฉพาะตัวเลขออกมา

บรรทัดตัวอย่าง:

```text
X-DSPAM-Confidence: 0.8475
```

โค้ด:

```python
pos = line.find(':')
str_val = line[pos+1:]
```

การทำงาน:

- `find(':')` หา index ของ `:`
- `pos+1` คือจุดเริ่มต้นหลัง `:`
- `line[pos+1:]` คือการตัดข้อความตั้งแต่หลัง `:` ไปจนจบ

ผลลัพธ์ที่ได้อาจเป็น:

```text
 0.8475\n
```

ซึ่งยังเป็น **สตริง** และยังมีช่องว่างหรือ `\n` ติดอยู่

#### 3. การแปลงชนิดข้อมูลด้วย `float()`

ค่าที่อ่านจากไฟล์จะเป็นข้อความเสมอ แม้มันจะดูเหมือนตัวเลขก็ตาม

ดังนั้นต้องแปลงจาก:

- `"0.8475"` -> สตริง

ให้เป็น:

- `0.8475` -> ตัวเลขทศนิยม

จึงใช้:

```python
val = float(str_val.strip())
```

โดย:

- `strip()` ตัดช่องว่างหน้า-หลังและ `\n`
- `float()` แปลงสตริงให้เป็นเลขทศนิยม

ถ้าไม่แปลงเป็น `float` เราจะเอาไปบวกทางคณิตศาสตร์ไม่ได้อย่างถูกต้อง

#### 4. การสะสมค่า (Accumulator Pattern)

บรรทัดนี้คือรูปแบบคลาสสิกของการสะสมผลรวม:

```python
total_val = total_val + val
```

แนวคิดคือ:

- เริ่มต้นผลรวมเป็น `0`
- ทุกครั้งที่เจอค่าที่ต้องการ ให้นำไปบวกเพิ่ม
- เมื่อจบลูป `total_val` จะเก็บผลรวมทั้งหมด

นี่เรียกว่า **accumulator pattern**

#### 5. การนับจำนวนข้อมูล (Counter Pattern)

บรรทัดนี้คือรูปแบบการนับ:

```python
count = count + 1
```

ใช้เพื่อบอกว่า:

- เราพบข้อมูลที่ตรงเงื่อนไขทั้งหมดกี่บรรทัด
- ต้องใช้จำนวนนี้เป็นตัวหารตอนคำนวณค่าเฉลี่ย

นี่เรียกว่า **counter pattern**

#### 6. การหาค่าเฉลี่ย

สูตรพื้นฐานคือ:

```python
average = total_val / count
```

แต่จะทำได้ก็ต่อเมื่อ `count > 0`

ถ้าไม่มีข้อมูลเลยแล้วเอาไปหาร จะเกิดข้อผิดพลาดแบบ **division by zero** ได้ จึงต้องเช็กก่อนเสมอ

### ลำดับการไหลของข้อมูล

`รับชื่อไฟล์` → `เปิดไฟล์` → `อ่านทีละบรรทัด` → `กรองเฉพาะบรรทัดที่ต้องการ` → `สกัดตัวเลข` → `แปลงเป็น float` → `สะสมผลรวม` → `นับจำนวน` → `หารเพื่อหาค่าเฉลี่ย`

### ตัวอย่างจำลองการทำงาน

สมมติในไฟล์มีบรรทัดต่อไปนี้:

```text
From: user@example.com
X-DSPAM-Confidence: 0.8475
X-DSPAM-Confidence: 0.765
Subject: Hello
X-DSPAM-Confidence: 0.9000
```

โปรแกรมจะสนใจเฉพาะ 3 บรรทัดนี้:

```text
X-DSPAM-Confidence: 0.8475
X-DSPAM-Confidence: 0.765
X-DSPAM-Confidence: 0.9000
```

แล้วคำนวณดังนี้:

- `total_val = 0.8475 + 0.765 + 0.9000 = 2.5125`
- `count = 3`
- `average = 2.5125 / 3 = 0.8375`

ผลลัพธ์จะประมาณ:

```text
Average spam confidence: 0.8375
```

### จุดเรียนรู้สำคัญจากตัวอย่างนี้

- ไฟล์ที่อ่านเข้ามาให้ข้อมูลเป็น **ข้อความ**
- เราสามารถกรองบรรทัดที่ต้องการด้วย `startswith()`
- เราสามารถหาตำแหน่งข้อมูลสำคัญด้วย `find()`
- เราสามารถตัดข้อความบางส่วนด้วย slicing เช่น `line[pos+1:]`
- ข้อมูลที่ดูเหมือนตัวเลขอาจยังเป็น string จึงต้องแปลงด้วย `float()`
- การหาค่าเฉลี่ยต้องใช้ทั้ง **ผลรวม** และ **จำนวนข้อมูล**
- `count` และ `total_val` เป็นตัวอย่างสำคัญของ **counter pattern** และ **accumulator pattern**

### ข้อสังเกตเพิ่มเติม

- ใช้ชื่อ `total_val` แทน `sum` เป็นเรื่องที่ดี เพราะ `sum` เป็นชื่อฟังก์ชันในตัวของ Python
- ถ้าบรรทัดเปลี่ยนรูปแบบ เช่น ไม่มี `:` โค้ดนี้อาจต้องเพิ่มการตรวจสอบมากขึ้น
- ในงานจริง อาจเขียนให้สั้นลงได้ แต่เวอร์ชันนี้เหมาะมากสำหรับการเรียนรู้ เพราะเห็นทุกขั้นตอนชัดเจน

## ✅ เช็คลิสต์ทบทวน (ก่อนเขียนโปรแกรมอ่าน/เขียนไฟล์)

- [ ] เข้าใจว่า **Handle** คือสะพาน ไม่ใช่ข้อมูลทั้งไฟล์
- [ ] รู้โหมด `'r'` (อ่าน) `'w'` (เขียนทับ) `'a'` (เพิ่มท้าย)
- [ ] ใช้ **`with open(...) as f`** เพื่อปิดไฟล์อัตโนมัติ
- [ ] ระบุ **`encoding='utf-8'`** เมื่อมีภาษาไทย
- [ ] รู้ว่า **`\n`** คืออักขระขึ้นบรรทัดใหม่ (หนึ่งตัว)
- [ ] กรองบรรทัดที่ต้องการด้วย `startswith()`
- [ ] สกัดค่าบางส่วนจากข้อความด้วย `find()` และ slicing
- [ ] แปลงข้อความเป็นตัวเลขด้วย `float()` ก่อนนำไปคำนวณ

**โฟลว์เริ่มต้น:** เปิดไฟล์ → อ่าน/เขียนผ่าน handle → ปิด (หรือใช้ with)

## 🎯 ลองทำ (แบบฝึกสั้น)

1. **สร้างไฟล์ `.txt`** แล้วเขียนหนึ่งบรรทัดด้วย `open(..., 'w')` และ `f.write(...)` จากนั้นเปิดอ่านด้วย `open(..., 'r')` และ `f.read()` แล้วพิมพ์ออกมา
2. **อ่านไฟล์ทีละบรรทัด** ด้วย `for line in f:` แล้วใช้ `line.strip()` ตัด `\n` ออก และพิมพ์แต่ละบรรทัด
3. **ดัดแปลงตัวอย่างด้านบน** ให้เปลี่ยนจาก `upper()` เป็น `lower()` แล้วสังเกตผลลัพธ์
4. **ลองพิมพ์ชื่อไฟล์ผิด** เพื่อดูว่า `try/except` ช่วยให้โปรแกรมจบแบบสุภาพแทนการขึ้น error ได้อย่างไร
5. **เขียนโปรแกรมหาผลรวม** ของค่า `X-DSPAM-Confidence` โดยยังไม่ต้องหารค่าเฉลี่ย
6. **เปลี่ยนเงื่อนไขการกรอง** จาก `startswith()` เป็นการค้นหาคำอื่นในไฟล์ แล้วพิมพ์เฉพาะบรรทัดที่ตรงเงื่อนไข
