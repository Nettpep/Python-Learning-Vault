# For Loop สะสมค่า (Accumulator Pattern)

**หมวด:** 20_Logic_Flow — Loops  
**วันที่บันทึก:** 2026-03-08  
**Tags:** `#learning` `#walkthrough` `#for-loop` `#list` `#accumulator`

---

## 🎯 สิ่งที่เรียนรู้

ชุดคำสั่งนี้ได้ผลลัพธ์เป็น **40** เพราะเป็นการทำงานแบบ **"สะสมค่า" (Accumulator Pattern)** โปรแกรมนำตัวเลขในรายการ (List) มาบวกเพิ่มเข้าไปในตัวแปร `running_total` ทีละตัวจนครบ

---

## 📝 โค้ดตัวอย่าง

```python
values = [5, 20, 7, 8]

running_total = 0
for number in values:
    running_total = running_total + number

print(running_total)  # 40
```

---

## 🔍 Trace — จำลองการทำงานทีละขั้นตอน

| รอบที่ | ค่าของ `number` | การคำนวณ (`running_total + number`) | ค่าใหม่ของ `running_total` |
|--------|-----------------|-------------------------------------|-----------------------------|
| เริ่มต้น | — | — | 0 |
| 1 | 5 | 0 + 5 | 5 |
| 2 | 20 | 5 + 20 | 25 |
| 3 | 7 | 25 + 7 | 32 |
| 4 | 8 | 32 + 8 | 40 |

---

## 💡 ทำไมถึงได้ผลลัพธ์นี้? (วิเคราะห์เหตุผล)

- **การประกาศค่าเริ่มต้น:** บรรทัด `running_total = 0` คือการเตรียม "ถังเปล่า" ไว้รอรับการบวกเพิ่ม
- **การวนซ้ำ (Iteration):** `for number in values:` สั่งให้ Python หยิบเลขจาก `[5, 20, 7, 8]` มาทีละตัวตามลำดับ
- **การอัปเดตค่า:** บรรทัด `running_total = running_total + number` หมายถึง "เอาค่าที่มีอยู่ในถังตอนนี้ บวกกับเลขตัวใหม่ แล้วเก็บผลลัพธ์กลับลงไปในถังเดิม"
- **ผลรวมสุดท้าย:** เมื่อบวกครบทุกตัว \(5 + 20 + 7 + 8\) ผลรวมที่ได้จึงเท่ากับ **40** พอดี

---

## 📌 ข้อสังเกตเพิ่มเติม

- แบบแผน Accumulator ใช้ได้กับทั้งการบวก (ผลรวม) การคูณ (ผลคูณสะสม) หรือการต่อสตริง (รวมข้อความ)
- ถ้าเปลี่ยน `values` เป็น list อื่น โปรแกรมจะคำนวณผลรวมของ list นั้นแทน
